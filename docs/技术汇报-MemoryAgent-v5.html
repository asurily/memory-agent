解决什么问题
AI Agent 的核心痛点：对话结束即遗忘，无法跨会话保持用户上下文
❌ 没有 Memory Agent
LLM 每次对话都从零开始，无法记住用户偏好
用户重复说"我是Java开发"、"用MySQL"等基础信息
无法从历史对话自动提取值得记忆的信息
记忆冲突无法处理（用户改了偏好但旧记忆还在）
对话越来越长，Token 消耗线性增长，无自动压缩
Prompt 硬编码在代码中，无法按业务领域调优
每接入一个场景都要重新实现记忆管理逻辑
✅ 使用 Memory Agent
跨会话记住用户画像、偏好、历史决策

v5 新特性：Prompt 策略可插拔
5 处 LLM prompt 全部通过 SPI 可定制，不同业务领域零改动接入

PromptStrategy SPI

系统中 5 处 LLM 交互的 prompt（意图总结、记忆提取、冲突检测、上下文压缩、增量意图）原本硬编码在代码中，
不同业务场景（医疗、客服、编程）无法调优。现在通过 PromptStrategy SPI 完全解耦：

• 默认 DefaultPromptStrategy 提供中文 prompt，开箱即用
• 继承并选择性覆盖任意方法，只需写差异部分
• 支持可选 system message，充分利用 LLM 的 system prompt 能力
• keyParams 可通过 YAML 配置，无需写代码

领域适配示例：医疗场景
医疗场景需要不同的记忆分类（symptom/diagnosis/medication/allergy）和意图参数（bodyPart/severity），
只需继承 DefaultPromptStrategy 覆盖 2 个方法，其余 3 个方法自动继承默认行为。
📝 Java API 注入
MemoryAgent agent = MemoryAgent.builder() .config(config) .promptStrategy(new MedicalPromptStrategy()) .build();
🌿 Spring Boot Bean
@Bean public PromptStrategy promptStrategy() { return new MedicalPromptStrategy(); } // 或仅配置 keyParams（零代码） # application.yml memory-agent: key-params-schema: | "symptom": "症状" "diagnosis": "诊断"

分层记忆架构
记忆自动分为三层，平衡 Prompt 效果与存储容量
L1: CORE 核心记忆
始终注入 Prompt • 用户画像 + Agent 笔记 • captureSnapshot() 只取这一层
↓ 衰退评分 < 阈值 / archiveMemory()
L2: ARCHIVED 归档记忆
语义检索 • 不占 Prompt 空间 • 衰退降级 / 手动归档的记忆
↓ 原始对话记录
L3: RAW 原始记录
按需检索 • 完整对话消息 • 可选
与其他功能的联动
• 记忆衰退 → 低分记忆自动降级到 ARCHIVED
• 容量检查 → 只计算 CORE 层记忆
• 快照捕获 → 只包含 CORE 层记忆
• 归档记忆 → 可通过语义搜索找回
• promoteMemory() → 手动提升回 CORE 层
自动管道编排
processUserMessage 和 recordAssistantMessage 两个方法自动编排多步操作
processUserMessage -- 一站式处理
1. 记录用户消息
2. 加载冻结记忆快照
3. 加载用户画像
4. 获取最近意图总结
5. 获取消息列表
6. 检查总结条件
7. 推导增量意图
8. 若条件满足 → ASYNC 后台总结
9. 组装 TurnContext 返回
recordAssistantMessage -- 自动压缩
1. 记录助手消息
2. 若压缩条件满足 → ASYNC 后台压缩

设计亮点
关键设计决策及其技术价值
设计决策实现方式价值
零框架依赖 core 仅依赖 SLF4J，手动 JSON 解析，java.net.http.HttpClient 可嵌入任意 Java 项目，无依赖冲突
不可变值对象 17 个 model 类全部 final + private constructor + getter only 天然线程安全，无并发问题
冻结快照 会话启动时捕获记忆快照，期间不变 保护 LLM Prefix Cache，减少 Token 消耗
能力自动启用 移除全部 10 个 boolean 开关，检测 apiKey 自动启用 极简接入，一个 apiKey 全部就绪
守护线程池 pipeline / decay / conflict 三个单线程守护线程 后台自动运行，不阻塞主流程
Prompt 可插拔 PromptStrategy SPI + 5 个 Context 类 + PromptTemplate 值对象 不同业务领域零改动接入，子类继承只写差异
ConcurrentHashMap + compute() 存储层原子操作，无锁并发 高并发场景下的线程安全保证
LongAdder 指标 MemoryMetricsCollector 使用 LongAdder 零 GC 压力的性能监控
