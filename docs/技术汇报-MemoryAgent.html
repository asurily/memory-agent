一、项目定位：AI Agent 的"记忆系统"

解决什么问题？

大模型本质是无状态的 —— 每次对话都是全新的开始。用户需要反复自我介绍，Agent 切换会话后丢失所有上下文。 这严重限制了 AI Agent 在客服、助手、陪练等场景的实用价值。

Memory Agent 填补了这个空白：它是一套完整的、可嵌入任何 Java 项目的记忆子系统。

适用场景

✅ 智能客服 — 记住用户历史问题与偏好
✅ 编程助手 — 记住技术栈、项目上下文
✅ AI 陪练 — 记住用户水平与进展
✅ 企业知识库 — 构建用户级长期画像
✅ 任何需要"跨对话连续性"的 LLM 应用

二、核心价值主张

极简接入

3 行代码即可拥有完整记忆能力。 零框架依赖（core 仅依赖日志门面 SLF4J）， 纯 Java SDK 可嵌入任何项目。Spring Boot 项目只需添加 Starter 依赖 + 几行配置。

渐进式增强

所有高级功能均为配置开关，默认关闭。 基础版 = 基础能力，打开开关 = 自动获得增强。 不开启 = 零性能开销 = 完全向后兼容。团队可以按需逐步启用。

全栈可插拔

存储、LLM、向量嵌入、向量数据库，全部通过 SPI 机制解耦。 内置默认实现（内存存储 + OpenAI 兼容 API），生产环境可无缝替换为 MySQL、Milvus、Claude 等。

工程级质量

不可变值对象 + ConcurrentHashMap 原子操作 + 守护线程模型。 全链路线程安全，自动资源回收。分层架构保证数据不丢失，衰退降级可找回。

安全第一

三层安全防护： 不可见 Unicode 检测 → 提示注入检测 → 数据泄露/后门检测。 所有记忆写入和 LLM 输入前自动扫描，安全异常阻断并报警。

开箱即用

默认实现覆盖全部功能路径。不需要对接任何外部服务就能跑通全流程。内置简易 Embedding（TF-IDF 哈希）， 内置内存向量存储，零外部依赖即可体验语义搜索。

六、技术设计亮点

1. 冻结快照：保护 LLM Prefix Cache

会话启动时捕获 CORE 层记忆快照并冻结。会话期间写入新记忆不影响当前会话的 Prompt，从而保护 LLM Prefix Cache。 新会话自动获取最新记忆。这解决了"记忆更新导致缓存失效"的核心矛盾。

2. 衰退 = 降级，不是删除

低分记忆不会被删除，而是降级到 ARCHIVED 层。不占用 Prompt 空间，但可通过语义搜索随时找回。 高重要性记忆（score ≥ 0.8）受永久保护。这保证了数据安全：记过的就不会丢，但不会挤占上下文。

3. SPI Default 方法扩展

所有新增 SPI 方法均使用 Java interface default 方法实现，返回合理默认值。已有实现完全不受影响， 无需修改任何代码。这实现了"不破坏即有实现的接口演进"，是插件架构的最佳实践。

4. 不可变值对象 + ConcurrentHashMap

所有核心数据模型（MemoryEntry、Session、ConversationMessage）均为 final + private constructor， 修改通过 with* 方法返回新实例。配合 ConcurrentHashMap 的 compute() 原子操作，实现全链路线程安全。

5. 三层安全防护

Layer 1: 不可见 Unicode 字符检测（零宽空格、RTL 覆盖等 7 种）→ Layer 2: 提示注入正则匹配（6 种模式） → Layer 3: 数据泄露/后门检测（curl 泄露、密钥读取、SSH 后门）。在所有记忆写入和 LLM 调用前自动执行。

6. 配置驱动的零开销设计

高级功能通过 boolean 配置开关控制。未开启的功能，其对应的组件根本不会被创建（Builder 中条件跳过）， 线程池不会被启动，向量计算不会执行。真正的零开销，不是"运行了但不生效"。

十、总结：为什么选 Memory Agent

对技术决策者

填补 Java AI 生态空白。 业界记忆方案集中在 Python（LangChain Memory、Mem0）， 几乎没有成熟的 Java 原生方案。Memory Agent 是目前 Java 生态中功能最完整的多轮对话记忆系统。

低成本验证，高上限扩展。 基础功能 3 行代码接入，高级功能配置开关逐步开启。 不需要一开始就投入大量工程资源。

对业务价值

AI Agent 从"能用"到"好用"的关键一步。 有了记忆的 Agent 不再是每次重新开始的"金鱼脑"，而是持续理解用户、积累上下文、个性化服务的"数字助理"。

直接提升核心指标： 上下文连贯性（用户无需重复自我介绍）、回复相关性（基于历史理解意图）、 用户留存（长期关系积累）。
